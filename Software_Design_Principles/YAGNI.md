# YAGNI Principle

## Simple Definition
YAGNI stands for "You Aren't Gonna Need It". It means do not build features or abstractions unless they are truly needed right now.

## Real-life Example
If your app currently needs only one payment method, do not build a large, flexible payment framework for future methods that may never be used.

## Why This Principle Matters
Building unnecessary features wastes time and adds complexity. It can make the system harder to understand and maintain.

## Benefits
- Faster delivery
- Less complexity
- Lower cost of maintenance
- Cleaner and focused code

## Common Mistakes When Violating YAGNI
- Adding features "just in case"
- Creating abstraction before real need
- Overbuilding for future requirements that may never come
